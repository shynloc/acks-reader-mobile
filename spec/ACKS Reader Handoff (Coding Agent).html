0 · First hour

Your first hour — do exactly this

Slice 0 → a file opens and renders

  1. Create an Android project: Kotlin, Jetpack Compose, minSdk 26, targetSdk 35.
  2. Copy render-core/ (host.html, markdown.js, themes.js, extensions.js) into app/src/main/assets/web/.
  3. Add the intent filters from §5 to AndroidManifest.xml so the app appears in "Open with".
  4. Implement the import controller (§6): on ACTION_VIEW / ACTION_SEND, copy the content:// stream into app sandbox, detect MD vs HTML.
  5. Add a full-screen WebView (§7), load host.html, then call ACKS.render(md, opts).
  6. Send yourself an .md in Telegram → "Open with ACKS Reader" → see it rendered in a theme.

When that round-trip works, you have the spine of the product. Everything else is additive (§9).

✱

Why this first: the spec's own directive — "implement in small vertical slices; first make the external-file-to-preview path work, then themes, then export, then polish." Do not start by building all screens.

2 · Architecture

Architecture: native shell + portable render core

One decision drives everything: documents render in the system WebView using the bundled JS core; all UI chrome is native Compose. The WebView shows only the document, never browser UI.

External app→ Intent filter→ Import controller→ Sandbox copy→ Format detect→ WebView · render core

Native (Kotlin / Compose)

Intents & import, sandbox storage, recent list, top bar, capsule toolbar, sheets (theme / viewport / export / safety / type / contents), settings, export pipeline, state machine, platform chrome (status bar insets, FAB, ripple).

WebView (bundled JS)

Markdown→HTML, 13 themes, Mermaid/math, gesture script (long-press / pinch-zoom / search / TOC). Driven by ACKS.render() / ACKS.command(); reports back via AndroidBridge.

▶

Agent directive: keep the shell neutral (grays + one orange accent), never compete with the document theme. Theme / viewport / type changes are just another ACKS.render() call with new opts — they must feel instant, with no reload flash.

5 · Intents

Intent filters & manifest

Register as an opener for .md, .markdown, .html, .htm via both ACTION_VIEW (open-with) and ACTION_SEND (share sheet). MIME types are unreliable across sender apps, so match both MIME and path/extension.

<!-- AndroidManifest.xml — inside <activity android:name=".PreviewActivity"> -->
<!-- 1) Open-with by MIME -->
<intent-filter>
  <action android:name="android.intent.action.VIEW"/>
  <category android:name="android.intent.category.DEFAULT"/>
  <category android:name="android.intent.category.BROWSABLE"/>
  <data android:mimeType="text/markdown"/>
  <data android:mimeType="text/html"/>
  <data android:mimeType="text/plain"/>
</intent-filter>
<!-- 2) Open-with by file extension (content:// & file:// & http) -->
<intent-filter android:autoVerify="false">
  <action android:name="android.intent.action.VIEW"/>
  <category android:name="android.intent.category.DEFAULT"/>
  <category android:name="android.intent.category.BROWSABLE"/>
  <data android:scheme="content"/>
  <data android:scheme="file"/>
  <data android:host="*"/>
  <data android:pathPattern=".*\\.md"/>
  <data android:pathPattern=".*\\.markdown"/>
  <data android:pathPattern=".*\\.html"/>
  <data android:pathPattern=".*\\.htm"/>
</intent-filter>
<!-- 3) Receive via share sheet -->
<intent-filter>
  <action android:name="android.intent.action.SEND"/>
  <category android:name="android.intent.category.DEFAULT"/>
  <data android:mimeType="text/*"/>
</intent-filter>

6 · Import

Import controller (content:// → sandbox)

External URIs can expire, so copy into the sandbox immediately, then always render the local copy. Detect format from extension first, then content sniff.

// ImportController.kt
data class ImportedDoc(
  val id: String, val title: String,
  val format: Format, val sourceApp: String,
  val sandboxPath: File, val sizeBytes: Long
)
enum class Format { MARKDOWN, HTML, UNSUPPORTED }

suspend fun import(ctx: Context, uri: Uri): ImportedDoc = withContext(Dispatchers.IO) {
  val name = queryDisplayName(ctx, uri) ?: "document"
  val id = UUID.randomUUID().toString().take(8)
  val dir = File(ctx.filesDir, "documents/$id").apply { mkdirs() }
  val ext = name.substringAfterLast('.', "").lowercase()
  val dest = File(dir, "source.$ext")
  ctx.contentResolver.openInputStream(uri).use { input ->
    dest.outputStream().use { input!!.copyTo(it) }
  }
  val fmt = when (ext) {
    "md", "markdown" -> Format.MARKDOWN
    "html", "htm"     -> Format.HTML
    else -> sniff(dest)   // <!doctype html> / <html → HTML, else MARKDOWN, else UNSUPPORTED
  }
  ImportedDoc(id, name, fmt, callerPackage(ctx), dest, dest.length())
}
✓ DoD

If the external URI expires after import, the sandbox copy still opens. Import errors show a useful message (→ unsupported / error state, §10).

7 · WebView

WebView host + native bridge

One WebView, full-bleed below the native top bar. Load host.html once; thereafter call ACKS.render(). JavaScript stays off for untrusted HTML in Safe Preview and on for Markdown / trusted HTML.

// PreviewWebView.kt — configuration
webView.settings.apply {
  javaScriptEnabled = true            // host.html needs JS to build the doc
  allowFileAccess = true
  builtInZoomControls = false          // pinch-zoom handled by the gesture script
  textZoom = 100
  cacheMode = WebSettings.LOAD_NO_CACHE
}
// bridge: document → native
webView.addJavascriptInterface(object {
  @JavascriptInterface fun onReady() { mainHandler.post { renderCurrent() } }
  @JavascriptInterface fun onMessage(json: String) { mainHandler.post { handleAcks(json) } }
}, "AndroidBridge")
webView.loadUrl("file:///android_asset/web/host.html")

// render / re-render (theme, viewport, type all funnel here)
fun renderCurrent() {
  val opts = JSONObject(mapOf(
    "themeId" to state.themeId, "mode" to state.mode,
    "interactive" to (state.format != HTML || state.htmlMode == "interactive"),
    "ov" to state.overridesJson
  )).toString()
  val src = JSONObject.quote(state.markdownSource)
  webView.evaluateJavascript("ACKS.render($src, ${JSONObject.quote(opts)})", null)
}

Native → document (search / jump / zoom-reset): webView.evaluateJavascript("ACKS.command('search', {\"q\":\"...\"})", null). Document → native arrives in onMessage(json) — parse __acks and update Compose state (search count, active heading, long-press menu position, zoom level). Full table in §10.

▶

Safety: for unknown HTML, default to Safe Preview — render with ACKS.renderHtml(html, /*safe*/true) (strips scripts) and show the "Enable interactive mode?" inline prompt. Full Interactive requires explicit user action ("Enable once" / "Always trust this file"). Never expose the AndroidBridge to untrusted document scripts — only the gesture script we inject uses it.

✓ DoD

Markdown renders through a theme; raw HTML renders in WebView; inline CSS/JS/SVG/Canvas work in Full Interactive; Safe Preview limits scripts; the WebView fills space and native controls never overlap content.

12.2 Import controller (Swift)

struct ImportedDoc {
  let id: String; let title: String
  let format: Format; let sandboxURL: URL; let size: Int64
}
enum Format { case markdown, html, unsupported }

func importDoc(_ src: URL) throws -> ImportedDoc {
  let scoped = src.startAccessingSecurityScopedResource()
  defer { if scoped { src.stopAccessingSecurityScopedResource() } }
  let id = String(UUID().uuidString.prefix(8))
  let dir = FileManager.default.urls(for: .documentDirectory, in: .userDomainMask)[0]
            .appendingPathComponent("documents/\(id)", isDirectory: true)
  try FileManager.default.createDirectory(at: dir, withIntermediateDirectories: true)
  let ext = src.pathExtension.lowercased()
  let dest = dir.appendingPathComponent("source.\(ext)")
  try FileManager.default.copyItem(at: src, to: dest)        // always render the local copy
  let fmt: Format = ext == "md" || ext == "markdown" ? .markdown
                  : (ext == "html" || ext == "htm" ? .html : sniff(dest))
  return ImportedDoc(id: id, title: src.lastPathComponent, format: fmt,
                     sandboxURL: dest, size: fileSize(dest))
}

12.3 WKWebView host + bridge

Load host.html from the bundle; drive it with evaluateJavaScript("ACKS.render(...)"). host.html calls window.AndroidBridge.onMessage(...); on iOS inject a tiny shim that maps that same name to WKScriptMessageHandler, so the host runs unchanged across platforms.

final class PreviewWeb: NSObject, WKScriptMessageHandler {
  let webView: WKWebView
  override init() {
    let cfg = WKWebViewConfiguration()
    let ucc = WKUserContentController()
    ucc.add(self, name: "acks")
    // shim so host.html's window.AndroidBridge.onMessage(json) reaches Swift
    let shim = """
      window.AndroidBridge = {
        onReady:   function(){ window.webkit.messageHandlers.acks.postMessage('{"__acks":"ready"}'); },
        onMessage: function(j){ window.webkit.messageHandlers.acks.postMessage(j); }
      };
    """
    ucc.addUserScript(WKUserScript(source: shim, injectionTime: .atDocumentStart,
                                   forMainFrameOnly: false))
    cfg.userContentController = ucc
    cfg.defaultWebpagePreferences.allowsContentJavaScript = true
    webView = WKWebView(frame: .zero, configuration: cfg)
    super.init()
    let host = Bundle.main.url(forResource: "host", withExtension: "html",
                              subdirectory: "web")!
    webView.loadFileURL(host, allowingReadAccessTo: host.deletingLastPathComponent())
  }
  func render(_ md: String, _ optsJSON: String) {
    webView.evaluateJavaScript("ACKS.render(\(md.jsQuoted), \(optsJSON.jsQuoted))")
  }
  func userContentController(_ c: WKUserContentController,
       didReceive msg: WKScriptMessage) {
    guard let json = msg.body as? String else { return }
    handleAcks(json)   // same __acks switch as Android §10
  }
}
▶

Agent directive · parity: the __acks message table (§10) and the DocState / Overrides contracts are platform-agnostic. Do not re-design them for iOS — reuse identically so the two shells stay behaviorally equal. Untrusted HTML: keep allowsContentJavaScript = false in Safe Preview; flip per-file only on explicit trust.
