ACKS Reader
UI / UX Specification

The complete interface & interaction spec for a mobile universal Markdown / HTML reader and exporter. Written for both design & product review and the coding AI agent that implements it.

Version 1.0 Platforms iOS 26 · Android 16 Accent Neutral shell + Orange #F26419 Themes 13 rendering themes Baseline ACKS Studio Design System
0 · Orientation

How to use this document

ACKS Reader solves one concrete problem: people receive .md / .html files inside WeChat, Feishu, Telegram and file managers, but have no previewer that is both beautiful and exports WYSIWYG. This app is the "Open with…" destination for those files.

For design / product readers

Parts 1–2 explain "what it looks like, how it moves" with diagrams and swatches. Each section is conclusion-first and stands alone.

For the coding AI agent

Part 3 gives copy-ready token constants, component props, the state machine, the message protocol and a build order. Paragraphs marked "Agent directive" are hard constraints.

▶

Agent directive · global: the app shell (nav, toolbar, panels) must stay neutral and visually quiet, giving all visual tension to the rendered document. The chrome must never compete with the document theme for color. Orange is for key-action accents only — never large fills.

1 · Principles

Design principles

Five principles run through the whole spec; on conflict, lower numbers win.

P1

Content is king, shell restrained

The UI is the frame, the document is the painting. Neutral grays, low contrast; never fight the theme.

P2

Native feel on both platforms

One IA, each platform per its latest HIG (large titles / sheet detents vs. FAB / ripple / M3 motion).

P3

Silky first

Every state change transitions; one easing cubic-bezier(0.16,1,0.3,1); gesture-driven, not button-only.

P4

WYSIWYG

Preview equals export. Viewport width, theme and type overrides are preserved 1:1 on export.

P5

Local-first & trustworthy

Files copied into a sandbox, never uploaded by default. Unknown HTML defaults to "Safe Preview"; scripts need explicit trust.

✱

Voice & copy: bilingual, neither language subordinate; no emoji; always numerals; "you" peer-to-peer. Inherits the ACKS Studio Design System voice.

5 · Motion

Motion

Motion is swift, precise, lightly springy — fitting a mechanical / tech tone. The primary easing comes from the ACKS Studio Design System.

Primary easing
cubic-bezier(0.16, 1, 0.3, 1) (fast out, subtle overshoot)
Micro
150–240ms — hover, press, toggle, segmented switch
Transition
300–420ms — sheet in/out, viewport scaling, theme switch
Page level
~500ms — large state changes
Press
scale(0.97); toggle knob uses cubic-bezier(0.34,1.4,0.5,1) for spring
Android ripple
600ms from touch point, opacity 0.55 → 0
▶

Agent directive · performance: animate only transform and opacity; avoid animating width/height/top (viewport scaling is the exception, via transition). Viewport switching scales the document iframe with transform: scale() rather than reflowing. Respect prefers-reduced-motion.

▶

Agent directive · scope: the Type panel is a preview setting, not an editor. It changes rendered presentation (CSS-variable overrides) only and never edits the source document. Overrides are injected as :root vars (--f-title/--f-body/--f-quote/--f-code/--f-table/--s-base/--c-accent) on the theme document.

10 · State Machine

State machine

Opening any document follows one lifecycle. The agent must implement every state, not just the happy path.

importing→ rendering→ rendered

Before rendering, branch to special / error states by document traits:

StateTriggerUI
loadingImport / first paintSkeleton + shimmer + spinner ("Importing / Rendering")
errorParse break / blank WebViewError page + "View source" fallback + re-import
largeVery large fileChunked render progress ("Rendered N/M chapters"), content after
unsupportedUnknown formatDedicated page + supported-format list + pick another
missingHTML refs local assets not bundledAmber banner "Missing N assets", placeholders in body

HTML safety mode

Unknown HTML defaults to Safe Preview (scripts & network paused); users can switch to Full Interactive (JS / Canvas / WebGL) and check "Always trust this file". Files always run sandboxed, with no access to other local files.

12.4 Build order (suggested)

  1. Build the markdown → HTML renderer and buildThemeDoc first (verify content renders).
  2. Implement 1–2 themes + theme switching; verify iframe isolation and scaling.
  3. Shell: device frame → top bar → capsule toolbar → recent list.
  4. Complete all 13 themes and sheet panels (theme / viewport / export / type).
  5. State machine (loading / error / large / unsupported / missing).
  6. Gestures (search / contents / pull-to-theme / long-press / zoom).
  7. Platform fork (iOS large title + detents; Android FAB + ripple).
  8. Export (print preview, WYSIWYG) + on-demand font loading.

12.5 Do / Don't

✓ Render docs in an isolated iframe ✓ flex/grid + gap layout ✓ One easing constant ✓ Full state-machine coverage ✓ Viewport via transform scale ✓ Type overrides via CSS vars
✕ Large orange / gradient fills in shell ✕ Parse huge docs sync on main thread ✕ Edit the source document ✕ Allow scripts on unknown HTML by default ✕ Emoji as icons ✕ Hit targets < 44pt
13 · Contracts & Protocol

Contracts & protocol

13.1 Document object

interface Doc {
  id: string;           // 'report.md'
  title: string;
  format: 'markdown' | 'html';
  source: string;       // origin app: Feishu / WeChat / Telegram …
  size: string;
  content: string;     // raw text (read-only)
  state?: 'ok'|'broken'|'large'|'unsupported'|'missing';
}

13.2 Overrides object

interface Overrides {
  fonts?: { title?, body?, quote?, code?, table?: string }; // css font-family
  baseSize?: number;     // px, body base
  colors?: { title?, body?, accent?: string };
}
// injected into theme :root: --f-title --f-body --f-quote --f-code --f-table --s-base --c-accent

13.3 iframe ⇄ shell message protocol

Every message carries an __acks field. Parent→iframe drives search / jump / zoom-reset; iframe→parent reports gesture results.

DirMessagePayload
P→Fsearch{q} highlight all matches
P→Ffindnav{dir:+1|-1} jump prev/next
P→FgotoHeading{i} scroll to heading i
P→FtrackHeadings / clearhl / resetzoomenable TOC tracking / clear highlight / reset zoom
F→Plongpress{word, x, top, bottom} menu position
F→Pzoom{zoom} current scale
F→Psearchres{count, cur} match count
F→Pactivehead{i} current heading index

13.4 Export

Export = rebuild a full document with the current theme + viewport width + type overrides and open a print preview (A4 uses @page A4; long-image / paginated use continuous page widths with break-inside: avoid on tables/code/callouts). WYSIWYG is mandatory: preview and export must be pixel-identical.

14 · Acceptance

Acceptance checklist

Verify item by item once implemented.

  • All 13 themes render tables, code, quotes, callouts, lists, Mermaid and math correctly in light & dark, recoloring per theme.
  • Theme / viewport switches transition smoothly (viewport via transform scale, no reflow flicker).
  • iOS: large title collapses on scroll; sheet half / full detents drag-snap; no FAB.
  • Android: list / settings rows ripple; bottom-right extended FAB; larger radii.
  • All five states covered; errors fall back to source; large docs show chunk progress.
  • Search highlight + count + jump; contents track-highlight + tap-to-jump.
  • Long-press menu, pinch / double-tap zoom (with reset badge), pull-to-theme.
  • HTML defaults to Safe Preview, switchable to Full Interactive with trust.
  • Type panel overrides fonts by role (on-demand Google Fonts) + size + accent, without editing source.
  • Export is WYSIWYG; shell stays neutral, orange only as accent.
  • Light/dark, accent, radius, font and toolbar variant switch live via Tweaks.
