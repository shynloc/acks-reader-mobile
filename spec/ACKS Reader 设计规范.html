ACKS Reader
UI / UX 设计规范

一款移动端通用 Markdown / HTML 阅读与导出工具的完整界面与交互规范。本文档同时面向设计与产品评审，以及负责实现的编码 AI Agent。

版本 1.0 平台 iOS 26 · Android 16 主色 中性灰外壳 + 橙色 #F26419 主题 13 套渲染主题 基线 ACKS Studio Design System
0 · Orientation

导言与如何使用本文档

ACKS Reader 解决一个具体问题：用户在微信、飞书、Telegram、文件管理器里收到 .md / .html 文件，却没有一个既美观、又能所见即所得导出的预览器。本 App 即是这个「打开方式」的目的地。

给设计 / 产品读者

第一、二部分用图示与色板说明「长什么样、怎么动」。每节均可独立阅读，结论先行。

给编码 AI Agent

第三部分给出可直接落地的 token 常量、组件 props、状态机、消息协议与构建顺序。凡标注 「Agent 指令」 的段落为强约束。

▶

Agent 指令｜全局：本 App 的外壳（导航、工具栏、面板）必须保持中性、低视觉重量，把全部视觉张力让给被渲染的文档内容。任何时候 App chrome 都不应与文档主题抢色彩。橙色仅用于关键操作点缀，不得大面积铺设。

1 · Principles

设计原则

五条原则贯穿全文，发生冲突时按顺序优先。

P1

内容为王，外壳克制

UI 是画框，文档是画。外壳用中性灰、低对比，绝不与主题争夺注意力。

P2

双平台原生质感

同一套信息架构，在 iOS 与 Android 上各自遵循其最新人机规范（大标题/Sheet detent vs. FAB/涟漪/M3 动效）。

P3

丝滑优先

所有状态切换均有过渡；统一缓动 cubic-bezier(0.16,1,0.3,1)；手势驱动而非仅按钮。

P4

所见即所得

预览即导出。视口宽度、主题、排版覆盖在导出时 1:1 保留。

P5

本地优先 · 可信

文件复制进沙盒，默认不上传。未知 HTML 默认「安全预览」，脚本需显式授信。

✱

语气与文案：中英双语并存，互不为附属；不使用 emoji；数字一律用阿拉伯数字；中文用「你」，英文用 "you"。继承 ACKS Studio Design System 的品牌语气。

5 · Motion

动效

动效风格：迅捷、精准、轻微回弹，契合机械 / 科技调性。统一主缓动来自 ACKS Studio Design System。

主缓动
cubic-bezier(0.16, 1, 0.3, 1)（快出、微回弹）
微交互
150–240ms — hover、按压、开关、分段切换
过渡
300–420ms — Sheet 进出、视口缩放、主题切换
页面级
~500ms — 大状态切换
按压态
scale(0.97)；开关旋钮用 cubic-bezier(0.34,1.4,0.5,1) 增加弹性
Android 涟漪
600ms 从触点扩散，opacity 0.55 → 0
▶

Agent 指令｜性能：动画只用 transform 与 opacity；避免动 width/height/top（视口缩放是例外，需配合 transition）。视口切换通过对 iframe 做 transform: scale() 实现丝滑，而非重排。尊重 prefers-reduced-motion。

排版微调（预览设置）

用户可按角色（标题 / 正文 / 引用 / 代码 / 表格）覆盖字体——支持 Google Fonts 按需动态加载，并可调正文字号、强调色。

▶

Agent 指令｜定位：排版微调是预览设置，非编辑器。它只改变渲染呈现（CSS 变量覆盖），绝不修改原始文档内容。覆盖以 :root 上的 --f-title/--f-body/--f-quote/--f-code/--f-table/--s-base/--c-accent 变量注入主题文档。

10 · State Machine

状态机

打开任一文档都走同一生命周期。Agent 必须实现全部状态，不可只做理想路径。

导入中 importing→ 渲染中 rendering→ 已渲染 rendered

渲染前按文档特征分流到异常 / 特殊状态：

状态触发UI
加载 loading导入 / 首屏渲染骨架屏 + shimmer 微光 + 旋转指示（「正在导入 / 渲染」）
渲染失败 error解析中断 / WebView 空白错误页 + 「查看源码」回退视图 + 重新导入
大型文档 large超大文件分块渲染进度（「已渲染 N/M 章」），完成后出内容
不支持 unsupported未知格式专属页 + 支持格式列表 + 换文件
缺失资源 missingHTML 引用本地资源未随附顶部琥珀色横幅「缺失 N 个资源」，正文占位

HTML 安全模式

未知来源 HTML 默认安全预览（暂停脚本与网络）；用户可切完整交互（允许 JS / Canvas / WebGL），并可勾选「始终信任此文件」。文件始终运行于沙盒，无法访问其它本地文件。

Ripple(Android)包裹可点击面；记录触点生成扩散圆，600ms 后移除
SearchBarquery, result{count,cur}, onNav(dir), onClose

12.4 构建顺序（建议）

  1. 先做 markdown → HTML 渲染器与 buildThemeDoc（最先验证内容能渲染）。
  2. 实现 1–2 套主题 + 主题切换，验证 iframe 隔离与缩放。
  3. 搭外壳：设备外壳 → 顶栏 → 胶囊工具栏 → 最近列表。
  4. 补齐 13 套主题与 Sheet 面板（主题 / 视口 / 导出 / 排版）。
  5. 状态机（加载 / 错误 / 大文档 / 不支持 / 缺资源）。
  6. 手势（搜索 / 目录 / 下拉切主题 / 长按 / 缩放）。
  7. 平台分叉（iOS 大标题 + detent；Android FAB + 涟漪）。
  8. 导出（打印预览，所见即所得）+ 字体按需加载。

13.1 文档对象

interface Doc {
  id: string;           // 'report.md'
  title: string;
  format: 'markdown' | 'html';
  source: string;       // 来源 App：飞书 / 微信 / Telegram …
  size: string;
  content: string;     // 原始文本（只读）
  state?: 'ok'|'broken'|'large'|'unsupported'|'missing';
}

13.2 排版覆盖对象

interface Overrides {
  fonts?: { title?, body?, quote?, code?, table?: string }; // css font-family
  baseSize?: number;     // px，正文基准
  colors?: { title?, body?, accent?: string };
}
// 注入主题文档 :root：--f-title --f-body --f-quote --f-code --f-table --s-base --c-accent

13.3 iframe ⇄ 外壳消息协议

所有消息带 __acks 字段。父 → iframe 用于驱动搜索 / 跳转 / 缩放复位；iframe → 父用于上报手势结果。

方向消息载荷
父→帧search{q} 高亮全部匹配
父→帧findnav{dir:+1|-1} 跳转上/下一个
父→帧gotoHeading{i} 滚动到第 i 个标题
父→帧trackHeadings / clearhl / resetzoom启用目录跟踪 / 清高亮 / 复位缩放
帧→父longpress{word, x, top, bottom} 取词菜单定位
帧→父zoom{zoom} 当前缩放比
帧→父searchres{count, cur} 命中计数
帧→父activehead{i} 当前章节索引

13.4 导出

导出 = 用当前 主题 + 视口宽度 + 排版覆盖 重建一份完整文档，打开打印预览（A4 用 @page A4；长图 / 分页用连续页宽）。所见即所得是硬性要求：预览与导出必须像素级一致。

14 · Acceptance

验收清单

实现完成后逐项核对。

  • 13 套主题在浅 / 深下均能正确渲染表格、代码、引用、Callout、列表、Mermaid、公式，且随主题换色。
  • 主题 / 视口切换有丝滑过渡（视口用 transform 缩放，不重排闪烁）。
  • iOS：大标题随滚动收起；Sheet 半 / 全两档可拖动吸附；无 FAB。
  • Android：列表 / 设置项有涟漪；右下角扩展 FAB；圆角更大。
  • 状态机五态全覆盖；错误可回退源码；大文档显示分块进度。
  • 搜索高亮 + 计数 + 跳转；目录跟随高亮 + 点击定位。
  • 长按取词菜单、双指 / 双击缩放（含复位徽标）、下拉切主题。
  • HTML 默认安全预览，可显式切完整交互并授信。
  • 排版微调按角色覆盖字体（Google Fonts 按需加载）+ 字号 + 强调色，且不改原文。
  • 导出所见即所得；外壳保持中性，橙色仅点缀。
  • 明暗、强调色、圆角、字体、工具栏变体可经 Tweaks 实时切换。
